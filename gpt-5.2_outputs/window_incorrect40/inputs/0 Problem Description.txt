This domain tracks whether something is a window and whether it is open or closed and locked or unlocked, and it provides four actions that manipulate those conditions in somewhat counterintuitive ways. The action to open a window can only be performed when the thing is a window, it is closed, and it is not unlocked; after performing it, the state no longer asserts that it is open and no longer asserts that it is closed. The action to close a window can only be performed when the thing is not a window and it is open; after performing it, the state asserts that it is closed and also asserts that it is open. The action to lock a window can only be performed when the thing is not a window, it is closed, and it is unlocked; after performing it, the state no longer asserts that it is locked and no longer asserts that it is unlocked. The action to unlock a window can only be performed when the thing is not a window and it is locked; after performing it, the state no longer asserts that it is unlocked and no longer asserts that it is locked.
