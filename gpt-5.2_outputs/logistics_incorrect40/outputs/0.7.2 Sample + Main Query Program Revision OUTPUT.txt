% [CHANGED] Replace inCarrier(package,vehicle) with two separate carrier fluents:
% NOTE: We keep the original declaration above for minimal edits, but we must not use loc(V) with V::vehicle.
% So we rewrite rule (2) into two rules, one for trucks and one for airplanes, and we also rewrite rule (1).

% 1. A package cannot be inside two different trucks at the same time.
impossible inCarrier(P,T1) & inCarrier(P,T2) & T1\=T2.

% 1b. A package cannot be inside two different airplanes at the same time.
impossible inCarrier(P,A1) & inCarrier(P,A2) & A1\=A2.

% 1c. A package cannot be inside a truck and an airplane at the same time.
impossible inCarrier(P,T) & inCarrier(P,A).

% 2. If a package is inside a truck, then the package's location is the same as the truck's location.
loc(P)=L if inCarrier(P,T) & loc(T)=L.

% 2b. If a package is inside an airplane, then the package's location is the same as the airplane's location.
loc(P)=L if inCarrier(P,A) & loc(A)=L.

% 3. Loading a package into a truck at a location causes the package to be inside that truck.
loadTruck(P,T,L) causes inCarrier(P,T).

% 4. Loading a package into a truck at a location causes the package to no longer be at that location.
% With functional loc/1, it is enough to set loc(P)=loc(T) via rule (2) once inCarrier holds.
% So we REMOVE the incorrect "keep other location" rule and instead require the truck is at L.
% (No direct loc(P) effect needed.)

% 5. Loading a package into a truck at a location is not executable if the truck is not at that location.
nonexecutable loadTruck(P,T,L) if loc(T)\=L.

% 6. Loading a package into a truck at a location is not executable if the package is not at that location.
nonexecutable loadTruck(P,T,L) if loc(P)\=L.

% 7. Loading a package into a truck at a location is not executable if the package is already inside some truck or airplane.
nonexecutable loadTruck(P,T,L) if inCarrier(P,T1).
nonexecutable loadTruck(P,T,L) if inCarrier(P,A1).

% 8. Loading a package into an airplane at a location causes the package to be inside that airplane.
loadPlane(P,A,L) causes inCarrier(P,A).

% 9. (Removed for same reason as truck case; location follows from inCarrier + rule 2b.)

% 10. Loading a package into an airplane at a location is not executable if the airplane is not at that location.
nonexecutable loadPlane(P,A,L) if loc(A)\=L.

% 11. Loading a package into an airplane at a location is not executable if the package is not at that location.
nonexecutable loadPlane(P,A,L) if loc(P)\=L.

% 12. Loading a package into an airplane at a location is not executable if the package is already inside some truck or airplane.
nonexecutable loadPlane(P,A,L) if inCarrier(P,T1).
nonexecutable loadPlane(P,A,L) if inCarrier(P,A1).

% 13. Unloading a package from a truck at a location causes the package to no longer be inside that truck.
unloadTruck(P,T,L) causes ~inCarrier(P,T).

% 14. Unloading a package from a truck at a location causes the package to be at that location.
unloadTruck(P,T,L) causes loc(P)=L.

% 15. Unloading a package from a truck at a location is not executable if the truck is not at that location.
nonexecutable unloadTruck(P,T,L) if loc(T)\=L.

% 16. Unloading a package from a truck at a location is not executable if the package is not inside that truck.
nonexecutable unloadTruck(P,T,L) if ~inCarrier(P,T).

% 17. Unloading a package from an airplane at a location causes the package to no longer be inside that airplane.
unloadPlane(P,A,L) causes ~inCarrier(P,A).

% 18. Unloading a package from an airplane at a location causes the package to be at that location.
unloadPlane(P,A,L) causes loc(P)=L.

% 19. Unloading a package from an airplane at a location is not executable if the airplane is not at that location.
nonexecutable unloadPlane(P,A,L) if loc(A)\=L.

% 20. Unloading a package from an airplane at a location is not executable if the package is not inside that airplane.
nonexecutable unloadPlane(P,A,L) if ~inCarrier(P,A).

% 21. Driving a truck from one location to another in a city causes the truck to be at the destination location.
driveTruck(T,L1,L2,C) causes loc(T)=L2.

% 23. Driving a truck from one location to another in a city is not executable if the truck is not at the starting location.
nonexecutable driveTruck(T,L1,L2,C) if loc(T)\=L1.

% 24. Driving a truck from one location to another in a city is not executable if the starting location is not in the given city.
nonexecutable driveTruck(T,L1,L2,C) if inCity(L1)\=C.

% 25. Driving a truck from one location to another in a city is not executable if the destination location is not in the given city.
nonexecutable driveTruck(T,L1,L2,C) if inCity(L2)\=C.

% 26. Flying an airplane from one location to another causes the airplane to be at the destination location.
flyPlane(A,L1,L2) causes loc(A)=L2.

% 28. Flying an airplane from one location to another is not executable if the airplane is not at the starting location.
nonexecutable flyPlane(A,L1,L2) if loc(A)\=L1.

% 29. Flying an airplane from one location to another is not executable if the starting location is not an airport.
nonexecutable flyPlane(A,L1,L2) if ~isAirport(L1).

% 30. Flying an airplane from one location to another is not executable if the destination location is not an airport.
nonexecutable flyPlane(A,L1,L2) if ~isAirport(L2).

% 31. It is not executable for a truck to drive from a location to the same location.
nonexecutable driveTruck(T,L1,L2,C) if L1=L2.

% 32. It is not executable for an airplane to fly from a location to the same location.
nonexecutable flyPlane(A,L1,L2) if L1=L2.

% 33. It is not executable to both load a package into a vehicle and unload the same package from that vehicle in the same step.
nonexecutable loadTruck(P,T,L) & unloadTruck(P,T,L1) if true.
nonexecutable loadPlane(P,A,L) & unloadPlane(P,A,L1) if true.

% 34. It is not executable to load the same package into two different vehicles in the same step.
nonexecutable loadTruck(P,T1,L1) & loadTruck(P,T2,L2) if T1\=T2.
nonexecutable loadPlane(P,A1,L1) & loadPlane(P,A2,L2) if A1\=A2.
nonexecutable loadTruck(P,T,L1) & loadPlane(P,A,L2) if true.

% 35. It is not executable to unload the same package from two different vehicles in the same step.
nonexecutable unloadTruck(P,T1,L1) & unloadTruck(P,T2,L2) if T1\=T2.
nonexecutable unloadPlane(P,A1,L1) & unloadPlane(P,A2,L2) if A1\=A2.
nonexecutable unloadTruck(P,T,L1) & unloadPlane(P,A,L2) if true.

Key reason the original outputs were all UNSAT: the program used `loc(V)` with `V :: vehicle`, but `loc/1` was declared as `loc(thing)` and the sorts were disjoint (no supersorts), so constraints like `loc(P)=L if inCarrier(P,V) & loc(V)=L` were ill-typed and effectively broke the intended model. The change above avoids using `loc(V)` for a generic `vehicle` by splitting the relevant constraints over `truck` and `airplane`.
