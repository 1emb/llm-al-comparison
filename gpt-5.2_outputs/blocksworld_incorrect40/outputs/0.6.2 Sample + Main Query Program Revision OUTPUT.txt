% And it is impossible for B2 to be clear while some block is on it.
impossible clear(B2) & loc(B1)=B2.

% 2. It is impossible for a block to be located on itself.
impossible loc(B)=B.

% 3. It is impossible for two different blocks to be holding in the hand at the same time.
impossible holding(B1) & holding(B2) & B1\=B2.

% 4-5. handEmpty and holding are mutually exclusive.
impossible handEmpty & holding(B).
impossible holding(B) & handEmpty.

% 6. Picking up a block causes the hand to hold that block.
pickup(B) causes holding(B).

% 7. Picking up a block causes the hand to not be empty.
pickup(B) causes ~handEmpty.

% 8. Picking up a block should remove it from its support (not set it to an arbitrary non-table location).

% 9. Picking up a block is not executable if the block is not clear.
nonexecutable pickup(B) if ~clear(B).

% 10. Picking up a block is not executable if the hand is not empty.
nonexecutable pickup(B) if ~handEmpty.

% 11. Picking up a block is not executable if the block is located on the table.
nonexecutable pickup(B) if loc(B)=table.

% 12. Putting down a block causes the hand to no longer hold that block.
putdown(B) causes ~holding(B).

% 13. Putting down a block causes the hand to become empty.
putdown(B) causes handEmpty.

% 14. Putting down a block causes the block to be located on the table.
putdown(B) causes loc(B)=table.

% 15. Putting down a block is not executable if the hand is not holding that block.
nonexecutable putdown(B) if ~holding(B).

% 16. Stacking a block onto another block causes the hand to no longer hold the moved block.
stack(B1,B2) causes ~holding(B1).

% 17. Stacking a block onto another block causes the hand to become empty.
stack(B1,B2) causes handEmpty.

% 18. Stacking a block onto another block causes the moved block to be located on the destination block.
stack(B1,B2) causes loc(B1)=B2.

% 19. Stacking a block onto another block causes the destination block to no longer be clear.
stack(B1,B2) causes ~clear(B2).

% (Optional but consistent) After stacking, the moved block is clear (nothing on top of it).
stack(B1,B2) causes clear(B1).

% 20. Stacking is not executable if the hand is not holding the moved block.
nonexecutable stack(B1,B2) if ~holding(B1).

% 21. Stacking is not executable if the destination block is not clear.
nonexecutable stack(B1,B2) if ~clear(B2).

% 22. Stacking is not executable if the moved block and the destination block are the same block.
nonexecutable stack(B1,B2) if B1=B2.

% 23. Unstacking a block from on top of another block causes the hand to hold the first block.
unstack(B1,B2) causes holding(B1).

% 24. Unstacking a block from on top of another block causes the hand to not be empty.
unstack(B1,B2) causes ~handEmpty.

% 25. Unstacking a block from on top of another block causes the second block to become clear.
unstack(B1,B2) causes clear(B2).

% (Optional but consistent) After unstacking, the moved block is clear.
unstack(B1,B2) causes clear(B1).

% 26. Unstacking is not executable if it is not the case that the first block is located on the second block.
nonexecutable unstack(B1,B2) if loc(B1)\=B2.

% 27. Unstacking is not executable if the first block is not clear.
nonexecutable unstack(B1,B2) if ~clear(B1).

% 28. Unstacking is not executable if the hand is not empty.
nonexecutable unstack(B1,B2) if ~handEmpty.

% 29-34. No concurrency between these action pairs.
