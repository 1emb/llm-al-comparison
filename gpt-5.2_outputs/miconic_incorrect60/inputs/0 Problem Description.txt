This domain models an elevator moving among floors and tracking people’s boarding and service status using a few basic facts: which entities count as floors, which entities count as passengers, where the lift currently is, which floor is above another, and for each person an origin floor, a destination floor, whether they have boarded, and whether they have been served. The boarding action can occur at a floor only when that location is a floor, the lift is currently at that floor, the person involved is not a passenger, and it is not the case that the person’s origin is that floor; when this boarding action happens, it makes the person not boarded by removing any record that they are boarded. The departing action can occur at a floor only when that location is a floor, the lift is currently at that floor, the person involved is not a passenger, it is not the case that the person’s destination is that floor, and it is not the case that the person is boarded; when this departing action happens, it marks the person as boarded and also marks the person as served. The upward movement action can occur between two floors only when the first location is not a floor, the second location is a floor, the lift is not at the first location, and the first location is above the second; when this upward move happens, it removes any record that the lift is at the second location and also removes any record that the lift is at the first location. The downward movement action can occur between two locations only when the first location is a floor, the second location is not a floor, the lift is at the first location, and it is not the case that the second location is above the first; when this downward move happens, it removes any record that the lift is at the second location and also removes any record that the lift is at the first location.
